# BoxTable 代码逻辑文档

## 组件架构

### 1. 组件关系图

```
┌─────────────────────────────────────────────────────────┐
│                   父组件（使用页面）                       │
│  ┌──────────────────────────────────────────────────┐  │
│  │                BoxTable（主组件）                  │  │
│  │  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌─────┐ │  │
│  │  │ QueryForm │ │TableToolbar│ │  Table   │ │Pagination│ │  │
│  │  └──────────┘ └──────────┘ └──────────┘ └─────┘ │  │
│  └──────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
```

### 2. 数据流

```
用户操作
    ↓
触发事件（搜索、重置、分页等）
    ↓
更新内部状态（searchParams、currentPage等）
    ↓
调用配置的 queryApi
    ↓
处理响应数据
    ↓
更新表格数据（tableData、total）
    ↓
重新渲染组件
```

#### 计算属性
```typescript
// 获取需要创建插槽的表格列
const tableSlotsColumns = computed(() => {
  return props.tableConfig.tableColumn?.filter((col: any) => 
    col.isSlot || col.type === 'expand'
  ) || []
})

// 显示的列（过滤隐藏的列）
const displayColumns = computed(() => {
  if (visibleColumns.value.length) {
    // 合并 selection 和 index 列
    const fixedColumns = props.tableConfig.tableColumn?.filter((col: any) =>
      col.type === 'selection' || col.type === 'index'
    ) || []
    return [...fixedColumns, ...visibleColumns.value]
  }
  return props.tableConfig.tableColumn || []
})
```

#### 核心方法

**数据加载 (`loadData`)**
```typescript
const loadData = async () => {
  if (!props.tableConfig.queryApi) return

  loading.value = true
  try {
    const params = {
      page: currentPage.value,
      pageSize: pageSize.value,
      ...searchParams.value
    }

    const res = await props.tableConfig.queryApi(params)

    if (res.code === 200) {
      tableData.value = res.data?.list || res.data || []
      total.value = res.data?.total || 0
      emit('load-success', { data: tableData.value, total: total.value })
    }
  } catch (error) {
    emit('load-error', error)
  } finally {
    loading.value = false
  }
}
```

**搜索处理 (`handleSearch`)**
```typescript
const handleSearch = (params: any) => {
  searchParams.value = params    // 更新查询参数
  currentPage.value = 1          // 重置到第一页
  loadData()                     // 加载数据
}
```

**列设置管理 (`handleColumnsChange`)**
```typescript
const handleColumnsChange = (columns: any[]) => {
  visibleColumns.value = columns  // 更新可见列
  
  // 保存到 localStorage
  if (props.tableConfig.tableKey) {
    localStorage.setItem(
      `${props.tableConfig.tableKey}_columns`,
      JSON.stringify(columns.map(col => col.prop || col.type))
    )
  }
}
```

#### 远程数据加载
```typescript
// 加载下拉选项数据
const loadSelectData = async (item: any) => {
  if (!item.queryApi || selectOptionsMap.value[item.field]) return
  
  selectLoadingMap.value[item.field] = true
  try {
    const res = await item.queryApi(item.queryParams || {})
    const options = item.formatData 
      ? item.formatData(res.data || res)
      : (res.data || res).map((d: any) => ({
          value: d[item.props?.value || 'value'],
          label: d[item.props?.label || 'label']
        }))
    
    selectOptionsMap.value[item.field] = options
  } finally {
    selectLoadingMap.value[item.field] = false
  }
}
```

### 3. 表格组件（tableOpe/index.vue）

#### 动态列渲染
```vue
<template>
  <el-table :data="tableData">
    <template v-for="col in tableColumn" :key="col.prop || col.type">
      <!-- 多选列 -->
      <el-table-column v-if="col.type === 'selection'" type="selection" />
      
      <!-- 序号列 -->
      <el-table-column v-else-if="col.type === 'index'" type="index" />
      
      <!-- 普通列 -->
      <el-table-column v-else :prop="col.prop" :label="col.label">
        <template #default="scope">
          <!-- 自定义插槽 -->
          <slot v-if="col.isSlot" :name="col.slotName" :row="scope.row" />
          
          <!-- 默认渲染（支持嵌套属性） -->
          <span v-else>{{ getValueByPath(scope.row, col.prop) }}</span>
        </template>
      </el-table-column>
    </template>
  </el-table>
</template>
```

#### 嵌套属性取值
```typescript
// 根据路径获取值（支持 a.b.c 格式）
const getValueByPath = (obj: any, path: string) => {
  if (!path) return ''
  const keys = path.split('.')
  let value = obj
  for (const key of keys) {
    if (value && typeof value === 'object') {
      value = value[key]
    } else {
      return ''
    }
  }
  return value
}
```

## 插槽系统详解

### 1. 插槽命名规则

#### 查询表单插槽
```typescript
// 配置：{ field: 'custom', isSlot: true, slotName: 'customField' }
// 使用：<template #query-customField>
```

#### 表格列插槽
```typescript
// 配置：{ prop: 'status', isSlot: true, slotName: 'status' }
// 使用：<template #status>
// 或：{ prop: 'status', isSlot: true } → <template #status>
```

### 2. 插槽作用域

#### 表格列插槽
```vue
<template #status="{ row, column, $index }">
  <!-- 
    row: 当前行数据
    column: 列配置对象
    $index: 行索引（0-based）
  -->
  <el-tag :type="row.status == 0 ? 'success' : 'danger'">
    {{ row.status == 0 ? '启用' : '停用' }}
  </el-tag>
</template>
```

#### 查询表单插槽
```vue
<template #query-customField="{ item, formData, updateFormData }">
  <!--
    item: 查询项配置
    formData: 当前表单数据
    updateFormData: 更新表单数据的函数
  -->
  <el-input 
    v-model="formData[item.field]" 
    @input="(val) => updateFormData(item.field, val)"
  />
</template>
```

### 3. 插槽传递机制

```typescript
// 主组件中动态创建插槽
<template 
  v-for="col in tableSlotsColumns" 
  :key="col.slotName || col.prop"
  #[getTableSlotName(col)]="scope"
>
  <slot 
    :name="getTableSlotName(col)" 
    :row="scope.row" 
    :column="col"
    :$index="scope.$index" 
  />
</template>

// 获取插槽名称
const getTableSlotName = (col: any) => {
  if (col.type === 'expand') return 'expand'
  return col.slotName || col.prop  // 直接使用列名，不加前缀
}
```

## 事件系统

### 1. 内部事件

#### 查询表单事件
- `@search`: 搜索按钮点击，传递查询参数
- `@reset`: 重置按钮点击

#### 表格事件
- `@selection-change`: 行选中状态变化
- `@sort-change`: 排序变化（待实现）

#### 工具栏事件
- `@refresh`: 刷新按钮点击
- `@export`: 导出按钮点击
- `@size-change`: 表格密度变化
- `@columns-change`: 列设置变化

#### 分页事件
- `@change`: 页码或页大小变化

### 2. 对外事件

#### 生命周期事件
- `@mounted`: 组件挂载完成，传递组件实例
- `@load-success`: 数据加载成功
- `@load-error`: 数据加载失败
- `@refresh`: 表格刷新
- `@export`: 数据导出

#### 数据事件
- `@selection-change`: 选中行变化

## 性能优化

```typescript

const loadSelectData = async (item: any) => {
  if (item.lazy && !selectOptionsMap.value[item.field]) {
```

### 2. 防抖搜索

```typescript
// 远程搜索防抖
const handleRemoteSearch = debounce(async (item: any, query: string) => {
  if (query.length < (item.minSearchLength || 2)) return
  await fetchRemoteOptions(item, query)
}, item.debounceTime || 300)
```

### 3. 数据缓存

```typescript
// 缓存远程数据
const fetchOptions = async (item: any) => {
  const cacheKey = `${item.field}_${JSON.stringify(item.queryParams || {})}`
  if (optionsCache[cacheKey]) {
    return optionsCache[cacheKey]
  }
  
  const options = await item.queryApi(item.queryParams || {})
  optionsCache[cacheKey] = options
  return options
}
```

## 总结

BoxTable 的代码逻辑清晰，模块化程度高，通过合理的组件划分和状态管理，实现了复杂表格功能的可配置化。其核心设计思想是：

1. **配置驱动**：通过 JSON 配置控制组件行为
2. **插槽扩展**：通过插槽系统支持自定义渲染
3. **状态集中**：主组件统一管理数据和状态
4. **事件通信**：通过事件系统实现组件间通信
5. **本地存储**：自动保存用户偏好设置

这种设计使得 BoxTable 既提供了丰富的开箱即用功能，又保持了高度的可扩展性。
